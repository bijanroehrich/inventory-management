Data Flow — End to End

1
Server Startup
mock_data.py runs json.load() on all 7 data files in server/data/. Python lists are held in memory for the lifetime of the process. No database involved.
2
User Applies Filters
FilterBar.vue stores selections in useFilters() composable — a shared reactive state (warehouse, category, order status, time period). Any change triggers a re-fetch across all views.
3
API Request
Vue views call api.js helpers (e.g. getInventory(filters)). Axios serializes filters as query params and sends a GET request to localhost:8001/api/inventory?warehouse=…&category=….
4
Backend Filtering
FastAPI endpoint calls apply_filters() or filter_by_month() on the in-memory list. Matching records are assembled into Pydantic model instances for type-safe serialization.
5
Response & Rendering
JSON is returned to Axios. The Vue view stores data in a ref(), triggers re-render. Search boxes apply additional client-side filtering via computed(). Clicking a row opens a detail modal via Teleport + Transition.
